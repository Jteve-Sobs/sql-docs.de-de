# <a name="odbc-driver-for-oracle-conformance-levels"></a>ODBC-Treiber für Oracle: Konformitätsgrade
> [!IMPORTANT]  
>  Diese Funktion wird in einer zukünftigen Version von Windows entfernt. Nutzen Sie diese Funktionen bei Neuentwicklungen nicht mehr, und planen Sie die Änderung von Anwendungen, die diese Funktion zurzeit verwenden. Verwenden Sie stattdessen den von Oracle bereitgestellten ODBC-Treiber.  
  
 ODBC definiert zwei Arten von Konformitäts Standards für Treiber: den API-Konformitäts Standard und den SQL-Grammatik Konformitäts Standard. API-Konformität bezieht sich auf die Funktionen, die von einem Treiber unterstützt werden. Die SQL-Übereinstimmung bezieht sich auf die SQL-Grammatik, die der Treiber unterstützt. Jeder Konformitäts Standard besteht aus Ebenen.  
  
 Dieser Abschnitt enthält Informationen zur Konformität des ODBC-Treibers für Oracle.
