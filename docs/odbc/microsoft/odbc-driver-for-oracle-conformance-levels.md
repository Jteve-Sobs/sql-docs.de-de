# <a name="odbc-driver-for-oracle-conformance-levels"></a>ODBC-Treiber für Oracle: Konformitätsgrade
> [!IMPORTANT]  
>  Diese Funktion wird in einer zukünftigen Windows-Version entfernt. Nutzen Sie diese Funktionen bei Neuentwicklungen nicht mehr, und planen Sie die Änderung von Anwendungen, die diese Funktion zurzeit verwenden. Verwenden Sie stattdessen den von Oracle bereitgestellten ODBC-Treiber.  
  
 ODBC definiert zwei Arten von Konformitätsstandards für Treiber: den API-Konformitätsstandard und den SQL-Grammatikkonformitätsstandard. API-Konformität bezieht sich auf die Funktionen, die ein Treiber unterstützt. SQL-Konformität bezieht sich auf die SQL-Grammatik, die der Treiber unterstützt. Jeder Konformitätsstandard besteht aus Ebenen.  
  
 Dieser Abschnitt enthält Konformitätsinformationen zum ODBC-Treiber für Oracle.  
  
 In diesem Abschnitt werden die folgenden Themen behandelt:

-   [Unterstützte Optionen (ODCB-Treiber für Oracle)](../../odbc/microsoft/supported-options-odbc-driver-for-oracle.md)  
  
-   [SQL-Konformitätsstufen (ODCB-Treiber für Oracle)](../../odbc/microsoft/sql-conformance-levels-odbc-driver-for-oracle.md)
