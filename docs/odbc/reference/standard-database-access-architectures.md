# <a name="standard-database-access-architectures"></a>Standardarchitekturen für den Datenbankzugriff
Betrachtet man die im vorherigen Abschnitt beschriebenen Datenbankzugriffskomponenten, so stellt sich heraus, dass zwei davon - Programmierschnittstellen und Datenstromprotokolle - gute Kandidaten für die Standardisierung sind. Die beiden anderen Komponenten - IPC-Mechanismus und Netzwerkprotokolle - befinden sich nicht nur auf einem zu niedrigen Niveau, sondern sie sind beide stark vom Netzwerk und Betriebssystem abhängig. Es gibt auch einen dritten Ansatz - Gateways -, der Möglichkeiten für die Standardisierung bietet.  
  
 In diesem Abschnitt werden die folgenden Themen behandelt:  
  
-   [Standardprogrammierschnittstelle](../../odbc/reference/standard-programming-interface.md)  
  
-   [Standardmäßiges Datenstromprotokoll](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [Standardgateway](../../odbc/reference/standard-gateway.md)
