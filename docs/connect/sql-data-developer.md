# <a name="sql-data-developer"></a>SQL Data Developer
Verwenden Sie die SQL-Datenplattform von Microsoft, um datenzentrierte Lösungen für mobile Geräte und Desktops für Webserver, Unternehmensserver und die Cloud zu erstellen.  

## <a name="sql-data-storage"></a>SQL-Datenspeicher
* [SQL Server Datenbank-Engine](../database-engine/configure-windows/sql-server-database-engine.md): Verwenden Sie den SQL Server Datenbank-Engine, um relationale Datenbanken für die Online Transaktionsverarbeitung oder analytische Online Verarbeitungsdaten zu erstellen. 
* [SQL Azure](https://docs.microsoft.com/azure/sql-database/): Verwenden Sie SQL Azure, um die Datenbank in die Cloud zu verschieben. 
* [SQL Compact](https://www.microsoft.com/download/details.aspx?id=17876): SQL Server Compact, um eigenständige und gelegentlich verbundene Anwendungen für mobile Geräte, Desktops und Webclients zu erstellen.
* [Localdb](../database-engine/configure-windows/sql-server-2016-express-localdb.md): Verwenden Sie localdb während der Entwicklung, um Transact-SQL-Code zu schreiben und zu testen, ohne eine vollständige Serverinstanz von SQL Server verwalten zu müssen.

## <a name="sql-data-tools"></a>SQL-Daten Tools
* [SQL Server Data Tools](../ssdt/download-sql-server-data-tools-ssdt.md) : Verwenden Sie SSDT zum Erstellen von relationalen Datenbanken, Azure SQL-Datenbanken, Integration Services Paketen, Analysis Services Datenmodellen und Reporting Services Berichten.
* [SQL Server-Verwaltungstools](../ssms/download-sql-server-management-studio-ssms.md): Verwenden von SSMS zum Konfigurieren, überwachen und Verwalten von Instanzen von SQL Server.

## <a name="sql-data-access"></a>SQL-Datenzugriff
* [SQL-Client Treiber](sql-connection-libraries.md): Verwenden Sie SQL-Treiber, um Daten aus einer SQL-Datenbank zu verbinden, abzufragen, zu aktualisieren, einzufügen oder zu löschen.
* [Entity Framework](https://msdn.microsoft.com/library/gg696172.aspx): Verwenden Sie die Entity Framework, um problemlos auf Datenbanken mit LINQ zuzugreifen, mit direktem Zugriff auf SQL Server und die Option zum Steuern der Zuordnung zwischen einer Entity Data Model (EDM) und unformatierten relationalen Strukturen. 
* [Windows Communication Foundation (WCF)](https://msdn.microsoft.com/library/dd456779.aspx): Verwenden Sie Data Services für eine nahezu sofort einsetzbare Lösung, um odata-Dienste problemlos im Web und Intranet zu erstellen und zu nutzen.
* [Synchronisierungs Framework](https://msdn.microsoft.com/library/jj839436.aspx): Verwenden Sie das Synchronisierungs Framework, um den Offline Zugriff auf jeden beliebigen Datentyp, jeden beliebigen Datenspeicher, jedes Übertragungsprotokoll und jede beliebige Netzwerktopologie zu aktivieren.
* [Reaktive Erweiterungen](https://msdn.microsoft.com/library/hh242985.aspx): Verwenden Sie die reaktiven Erweiterungen (RX), um die ereignisstreamprogrammierung auszuführen und die asynchrone Programmierung mithilfe von Observable-Sequenzen und LINQ-Stil Abfrage Operatoren zu vereinfachen.  Verwenden Sie die reaktiven Erweiterungen für JavaScript (rxjs), um die asynchrone Rückruf-und ereignisgesteuerte Programmierung im Web mit Observable-Sequenzen zu vereinfachen.
* [CLR-Integration](../relational-databases/clr-integration/common-language-runtime-clr-integration-programming-concepts.md): Verwenden Sie die CLR-Integration, um gespeicherte Prozeduren, Trigger, benutzerdefinierte Typen, benutzerdefinierte Funktionen, benutzerdefinierte Aggregate und Streaming-Tabellenwertfunktionen in einer beliebigen .NET Framework-Sprache zu schreiben, einschließlich Microsoft Visual Basic .NET und Microsoft Visual C#. 
* [SQLXML 4,0](../relational-databases/sqlxml/sqlxml-4-0-programming-concepts.md): Verwenden Sie SQLXML 4,0 zum Exportieren von relationalen Daten in XML.
