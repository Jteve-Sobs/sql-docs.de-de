# <a name="sql-server-help-and-feedback"></a>SQL Server-Hilfe und -Feedback
[!INCLUDE[appliesto-ss-asdb-asdw-pdw-md.md](../includes/appliesto-ss-asdb-asdw-pdw-md.md)]

Wir sind an Ihren Ideen zur Verbesserung unserer Produkte und Dokumentation interessiert. Wir können nicht immer persönlich Antworten, aber wir stellen sicher, dass Ihr Feedback an die richtige Person bzw. das zuständige Team weitergeleitet wird. 

## <a name="i-want-to-give-feedback-on-sql-server"></a>Ich möchte Feedback zu SQL Server abgeben
Das SQL Server-Technikteam hält ein Auge auf dem SQL Server[Benutzerfeedback](https://feedback.azure.com/forums/908035-sql-server). Wenn Sie ein Problem mit SQL Server melden möchten, z. B. einen Absturz, Leistungseinbußen, unerwartetes Verhalten usw., ist das Benutzerfeedback der richtige Ort hierfür. 

Das [Benutzerfeedback](https://feedback.azure.com/forums/908035-sql-server) ist außerdem ein hervorragender Ort, um dem Technikteam mitzuteilen, was es besser machen kann. Wenn Sie Empfehlungen oder Vorschläge für Funktionen haben, teilen Sie diese ebenfalls hier mit. 

## <a name="i-want-to-give-feedback-about-sql-server-documentation"></a>Ich möchte Feedback zur SQL Server-Dokumentation abgeben
Die beste Möglichkeit, das Team für SQL Server-Inhalte zu kontaktieren, ist über Git-Probleme. Sie finden am unteren Rand der meisten unserer Seiten eine Option, um ein Git-Problem zu übermitteln: 

![Inhaltsfeedback per „Git-Problem“](media/sql-server-get-help/git-issues.png)

Wenn Sie etwas anderes mitteilen möchten – eine Idee, einen Vorschlag, Feedback irgendetwas. Dann können Sie dies, indem Sie ein allgemeines Git-Problem unter [https://aka.ms/SQLDocsFeedback](https://aka.ms/SQLDocsFeedback) eintragen. 

Wählen Sie einfach die Schaltfläche „Neues Problem“ aus, um loszulegen. 

![Neues Git-Problem](media/sql-server-get-help/new-git-issue.png)


Das Team für SQL Server-Inhalte bittet auch häufig um Umfragefeedback. Wenn Sie etwas Zeit haben, teilen Sie uns bitte Ihre Erfahrungen unter [https://aka.ms/SQLDocSurvey](https://aka.ms/SQLDocSurvey) mit.


## <a name="i-want-to-make-a-suggestion-about-sql-server-management-studio"></a>Ich möchte einen Vorschlag zu SQL Server Management Studio machen
Das SQL Server Management Studio (SSMS)-Team überwacht ebenfalls eng das SQL Server-[Benutzerfeedback](https://feedback.azure.com/forums/908035-sql-server). Wenn Sie also Probleme zu melden oder Vorschläge zu unterbreiten haben, ist das Benutzerfeedback der richtig Ort hierfür. 

## <a name="i-want-to-make-a-suggestion-about-azure-data-studio"></a>Ich möchte einen Vorschlag zu Azure Data Studio machen
Das Azure Data Studio-Team überwacht umfassend Probleme und Featureanforderungen auf [Azure Data Studio Github](https://github.com/microsoft/azuredatastudio/issues). Reichen Sie ein Problem oder eine Featureanforderung hier ein. Es wird angezeigt, wenn das Problem behoben bzw. das Feature hinzugefügt wurde.

## <a name="i-want-to-access-the-sql-server-technical-documentation-offline"></a>Ich möchte auf die technische Dokumentation zu SQL Server offline zugreifen
Wenn Sie die SQL Server-Dokumentation auf einem Offlinesystem anzeigen möchten, haben Sie zwei Möglichkeiten. Sie können entweder eine PDF-Datei erstellen, wo immer Sie sich in der technischen Dokumentation zu SQL Server befinden, oder Sie können die Offlineinhalte mit dem [SQL Server Offline Help Viewer](sql-server-help-installation.md) herunterladen. 

Wenn Sie eine PDF-Datei erstellen möchten, klicken Sie auf den Link **PDF herunterladen**, der sich unten in jedem Inhaltsverzeichnis befindet.
![PDF herunterladen](media/sql-server-get-help/download-pdf.png)

## <a name="i-want-to-contact-microsoft-support"></a>Ich möchte den Microsoft-Support kontaktieren

Wenn Sie ein kommerzieller Benutzer sind, sehen Sie auf der Microsoft-Seite [Kontakt](https://support.microsoft.com/gp/contactus81?forceorigin=esmc&Audience=Commercial) nach.

## <a name="i-want-to-contribute-to-sql-server-documentation"></a>Ich möchte an der SQL Server-Dokumentation mitwirken

### <a name="edit-contentincludesmediaedit-topic-pencilpng-editing-content"></a>![Inhalt bearbeiten](../includes/media/edit-topic-pencil.png) Bearbeiten von Inhalt
Haben Sie Fehler in unserer Dokumentation gefunden? Oder haben Sie einen Beitrag, der unsere Dokumentation verbessern kann? Sie können den Inhalt von docs.microsoft.com selbst bearbeiten. Wenn Sie dies tun, wird unsere Dokumentation nicht nur besser, sondern Sie werden auch als Mitwirkender an der Seite aufgeführt. Weitere Informationen finden Sie unter [https://aka.ms/editsqldocs](https://aka.ms/editsqldocs). 

Alle öffentlichen Beiträge werden vom SQL Server-Inhaltsteam auf technische Richtigkeit und Konsistenz überprüft.
